Bingo (Connect 4) Game
• https://papergames.io/en/connect4
• Free game play site
• https://en.wikipedia.org/wiki/Connect_Four
• Detailed game rule explanation
• Connect Four is a classic two-player strategy game played on a vertical 7×6 grid.
• Objective: Be the first player to form a horizontal, vertical, or diagonal line of four of your own 
colored tokens.
• Gameplay: Players take turns dropping one token into any of the seven columns. The tokens fall 
straight down, occupying the lowest available space in that column.
• Ending the Game: The game ends when one player connects four in a row or when the board is 
completely filled, resulting in a draw.

Bingo Disqualification Condition
• Execution Error (Exception)
• If the player's decision method raises any Python exception 
• (e.g., a crash, syntax error within the logic, or reference error), 
• they are immediately disqualified
• Time Limit Exceeded (Timeout)
• If the time taken to return a move exceeds the defined floatTimeLimit (default is 5.0 seconds)
• Invalid Return Type
• The player must return two integers (y, x). 
• If the returned values are any other type (like strings, floats, or None), it results in disqualification. 
• Illegal Move (Feasibility)
• The move is disqualified if setState(x, y) returns False. 
• This happens if the move violates the rules of Connect Four defined
• You can use the game engine to identify this, but you should not inject any information to the game 
engine while checking the feasibility
• Out of Bounds: The coordinates are outside the grid.
• Occupied Space: The chosen cell already contains a token.
• Gravity Violation: The token is placed in mid-air.

Sample Agent Structure and Code
• PlayerAgent_99328162: Random Agent
• Identifying Legal Moves
• The decision method scans the board column 
by column (x). 
• For each column, it searches from the bottom 
row upwards (range(height - 1, -1, -1)) to find 
the first empty slot (0).
• Gravity Simulation: By breaking the inner loop 
after finding the first empty cell, it ensures it 
only considers the lowest available row in each 
column, correctly following the "gravity" rule of 
the game.
• Stochastic Strategy: It collects all these valid (y, 
x) coordinates into a list called feasible_moves. 
Then, it simply uses random.choice() to pick 
one of those moves at random.
• Terminal State: If the board is full and no moves 
are found, it returns (None, None), which, 
based on your GameEngine logic, would trigger 
a disqualification or an end-of-game state.
import random
from hw2.players.BingoPlayerAgent import BaseBingoPlayer
class PlayerAgent_99328162(BaseBingoPlayer):
def __init__(self, strPlayerName):
super().__init__(strPlayerName)
def decision(self, matGrid):
height, width = matGrid.shape
feasible_moves = []
for x in range(width):
for y in range(height - 1, -1, -1):
if matGrid[y][x] == 0:
feasible_moves.append((y, x))
break 
if feasible_moves:
return random.choice(feasible_moves)
return (None, None)

• Leaf node
• Requires some heuristics of scoring mechanism
• Loose case : -999999 score
• Winning case : 100 score
• # of Opponent’s 3 consecutive stones : -10 score for each case
• # of My 3 consecutive stones : 10 score for each case
• # of Opponent’s 2 consecutive stones : -3 score for each case
• # of My 2 consecutive stones : 3 score for each case
• Other node
• If it is my turn, I am going to maximize the reward, so find maximum reward from 
the node’s children
• If it is opponent’s turn, they will minimize the reward, so find minimum reward 
from the node’s children
